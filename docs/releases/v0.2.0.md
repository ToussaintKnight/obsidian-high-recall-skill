# v0.2.0

## Release Summary

This release turns the initial high-recall prototype into a more complete, independently testable tool for Obsidian users, researchers, and AI-agent workflows. It focuses on a no-private-vault first run, stronger packaging and diagnostics, broader verification, and a clearer contributor path.

Project page:

https://toussaintknight.github.io/obsidian-high-recall-skill/

Repository:

https://github.com/ToussaintKnight/obsidian-high-recall-skill

## What Is New

- Added a one-command fixture demo that does not require an Obsidian vault:

  ```bash
  npx --yes github:ToussaintKnight/obsidian-high-recall-skill demo
  ```

- Added a GitHub Codespaces path for evaluating the public fixture in a browser-based sandbox.
- Expanded the public fixture smoke suite to five English, Chinese, and mixed-language recall cases.
- Added privacy-safe `doctor --json` diagnostics, documented output contracts, CLI references, recipes, examples, and troubleshooting guidance.
- Added installed-package, package-content, binary, Markdown-link, and release-readiness checks.
- Expanded CI across Linux, Windows, and macOS.
- Added CodeQL, OpenSSF Scorecard, dependency maintenance, and stricter workflow credential handling.
- Added AI-readable project metadata, improved discovery topics, stable contribution entrypoints, and tester feedback paths.
- Added repeatable release, repository setup, launch metrics, issue, and Discussion workflows.

## Try It

Run the no-vault demo:

```bash
npx --yes github:ToussaintKnight/obsidian-high-recall-skill demo
```

Validate a local clone:

```bash
git clone https://github.com/ToussaintKnight/obsidian-high-recall-skill.git
cd obsidian-high-recall-skill
npm test
```

Query your own vault:

```bash
npx --yes github:ToussaintKnight/obsidian-high-recall-skill query "your broad research query" --vault "/absolute/path/to/your-vault" --backend auto --limit 120
```

## Privacy Notes

Vault notes, Smart Connections indexes, local OHS databases, raw queries, and recall packs remain on the user's machine unless the user explicitly shares an output. The public tests and demo use synthetic fixture notes. Do not attach private paths, snippets, labels, queries, `.smart-env`, `raw_runs.json`, or local cases files to issues or pull requests.

See [SECURITY.md](../../SECURITY.md) and the [privacy threat model](../privacy_threat_model.md).

## Limitations

- The private-vault benchmark is deployment evidence from one domain-skewed snapshot, not a universal retrieval benchmark.
- Smart Connections recall requires a compatible local `.smart-env`; behavior can vary across plugin versions and embedding models.
- OHS fallback can be slow on first use because it may build an index or download local dependencies and models.
- Codespaces and the public demo exercise synthetic fixtures, not private-vault discovery.
- Distribution is GitHub-backed for now; the npm registry command is not advertised until publication is independently verified. Readiness and the guarded workflow are documented in [docs/npm_publish.md](../npm_publish.md).
