# Roadmap

## v0.1: Make The Project Installable And Trustworthy

- Publish reproducible fixture-vault smoke tests.
- Keep the README focused on user pain, quickstart, privacy, and benchmark limits.
- Add CI, issue templates, contribution rules, and security policy.
- Make evaluator output match reported benchmark conditions, including `rrf-union`.

## v0.2: Improve Cross-Vault Reliability

- Collect public benchmark reports from different vault sizes and operating systems.
- Add stronger tests for bilingual queries, Smart Connections cache variants, and OHS fallback behavior.
- Keep [docs/troubleshooting.md](docs/troubleshooting.md) current for missing Smart vectors, stale OHS indexes, and large vaults.

## v0.3: Distribution

- Publish an npm package if external testers confirm the GitHub install path works reliably.
- Add a short demo video or GIF using only the public fixture vault.
- Prepare launch posts for Obsidian, local-first AI, PKM, and agent-tooling communities.
